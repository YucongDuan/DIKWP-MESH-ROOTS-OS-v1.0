# DIKWP-MESH² ROOTS OS v1.0

**No-definition semantic inquiry, distributed-purpose routing and project de-conceptualization runtime.**

ROOTS does not define *life*, *consciousness*, *intention*, *self*, *experience*, or any other named concept as an observer-free essence. It treats named concepts as temporary handles for observer/context/purpose-indexed semantic projections, then explores the full co-equal `DIKWP × DIKWP` transformation mesh.

## What it does

- registers all 25 ordered `D/I/K/W/P → D/I/K/W/P` primitive transformations;
- accepts incomplete, imprecise and inconsistent semantic resources;
- quarantines universal-definition claims without deleting them;
- routes semantic inquiry under a distributed, revisable purpose field;
- mines cross-frame invariant candidates while preserving branches and obstructions;
- proposes temporary new axes only with discriminator and rollback rules;
- compiles task-indexed semantic contracts;
- blocks new projects when they exhibit definition capture, hierarchy leakage, purpose monarchy, residual erasure or zero testability;
- maps a large GitHub portfolio as branches of shared root problems rather than a stack of named ontologies.

## Quick start

```bash
python -m venv .venv
source .venv/bin/activate
pip install -e '.[dev]'

dikwp-roots analyze examples/duan_repo_mission_bundle.json \
  --snapshot data/github_public_snapshot_2026-07-18.json \
  --out outputs/run

pytest -q
```

Open `outputs/run/dashboard.html` locally.

## Scientific boundary

The runtime produces auditable semantic bundles, not metaphysical certificates or final definitions. Its deterministic scores are routing and governance aids for a declared task, not measurements of an object's intrinsic essence.
